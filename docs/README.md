## Why the cleanup was done
The app accumulated multiple planning documents as the hackathon scope evolved. That was useful during execution, but it made the docs noisy and harder to navigate. The active docs now focus on:
- what the app is
- what is actually implemented
- how to run it
- how to deploy it
- how the product story developed over time
